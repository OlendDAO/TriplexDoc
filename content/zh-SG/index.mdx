# SWR

<Features lang={props.params.lang} title={metadata.title} />

"SWR" 名称来源于 `stale-while-revalidate`，这是一种由 [HTTP RFC 5861](https://tools.ietf.org/html/rfc5861) 推广的 HTTP 缓存失效策略。SWR 是一种策略，它首先从缓存返回数据（陈旧的），然后发送请求（重新验证），最终返回最新的数据。

<Callout emoji="✅">
  通过 SWR，组件将获得一个**持续**且**自动更新**的数据流。

用户界面将始终保持**快速**且**响应式**。

</Callout>
